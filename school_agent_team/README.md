# 학교팀 AI 에이전트 시스템

시험 합격을 위한 단기 튜터가 아니라, 특정 분야에서 실제로 이해하고 만들 수 있는 수준까지 성장하도록 돕는 CLI 기반 AI 에이전트 팀입니다.

예를 들어 전자기기를 뜯어보며 내부 구조와 작동 원리를 이해하고, 필요한 부품을 골라 간단한 장치를 직접 설계·제작하는 학습 흐름을 지원합니다.

## 1. 프로젝트 소개

이 프로젝트는 OpenAI Agents SDK를 사용해 여러 역할의 학습 에이전트를 구성합니다.

기본 동작은 두 가지입니다.

- 매니저 중심 답변: 간단한 질문은 학교팀 매니저가 직접 답변합니다.
- 팀 회의형 답변: 복잡한 요청은 여러 전문가 에이전트가 순차적으로 의견을 낸 뒤, 학교팀 매니저가 최종 정리합니다.

## 2. 6개월 전문가 성장형 학습 방식

학교팀은 사용자의 목표를 단기 암기 계획이 아니라 6개월 기준의 전문가 성장 과정으로 설계합니다.

기본 로드맵은 다음 흐름을 따릅니다.

1. 1개월차: 기초 개념 이해
2. 2개월차: 기본 구조와 작동 원리 이해
3. 3개월차: 기초 제작 및 실습
4. 4개월차: 응용 제작 및 문제 해결
5. 5개월차: 실제 제품 분석 및 재현
6. 6개월차: 개인 프로젝트 완성 및 포트폴리오화

단, 모든 답변을 무조건 긴 로드맵으로 만들지는 않습니다. 사용자가 장기 계획을 요청하거나 처음 목표를 말했을 때는 6개월 로드맵을 제시하고, 일상적인 질문은 현재 학습 단계와 연결해 짧고 실용적으로 답합니다.

## 3. 학습 흐름 유지 방식

학습 중간에 사용자가 관련 질문을 하면 학교팀은 질문에 답하되 전체 목표에서 벗어나지 않도록 안내합니다.

예를 들어 전자기기 학습 중 "커패시터가 뭐야?"라고 물으면 단순 정의만 말하지 않고, 현재 전자기기 구조 이해 단계에서 필요한 수준, 실제 제품에서의 쓰임, 나중에 더 깊게 배울 내용을 함께 알려줍니다.

## 4. 설치 방법

```bash
cd school_agent_team
python -m venv .venv
```

Windows PowerShell:

```powershell
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

macOS/Linux:

```bash
source .venv/bin/activate
pip install -r requirements.txt
```

## 5. .env 설정 방법

`.env.example` 파일을 참고해 `.env` 파일을 만듭니다.

```env
OPENAI_API_KEY=your_openai_api_key_here
```

## 6. 실행 방법

```bash
python main.py
```

실행하면 다음 안내가 출력됩니다.

```text
학교팀 AI 에이전트가 시작되었습니다.
배우고 싶은 주제나 목표를 입력하세요.
종료하려면 exit 또는 quit을 입력하세요.
```

## 7. CLI 명령어

```text
/archive on
/archive off
/archive status
/roadmap
/help
```

- `/archive on`: 학습 기록 저장을 켭니다. 기본값은 on입니다.
- `/archive off`: 학습 기록 저장을 끕니다.
- `/archive status`: 현재 저장 상태를 확인합니다.
- `/roadmap`: 현재 학습 목표 기준으로 6개월 전문가 성장 로드맵을 요청합니다.
- `/help`: 사용 가능한 명령어를 보여줍니다.

## 8. 사용 예시

```text
전자기기를 뜯어보면 이해하고 직접 만들 수 있는 수준이 되고 싶어. 어디서부터 시작해야 해?
```

```text
무선 충전기가 어떻게 작동하는지 학교팀 전체가 전문가처럼 분석해줘.
```

```text
고장난 선풍기를 뜯어서 원리를 이해하고 싶어.
```

## 9. 학교팀 구성원

- 학교팀 매니저: 목표 파악, 답변 방향 결정, 최종 학습 안내 정리
- 학습 설계자: 6개월 기준 초급 -> 중급 -> 고급 로드맵 설계
- 개념 교수: 어려운 개념을 쉬운 비유와 예시로 설명
- 원리 해석가: 실제 제품과 현상의 작동 원리 분석
- 분해/분석 전문가: 내부 구조, 부품 역할, 고장 가능 부위 분석
- 실습 코치: 준비물, 비용, 난이도, 단계별 실습 안내
- 설계 엔지니어: 회로, 구조, 부품 구성, 블록 다이어그램 제안
- 프로젝트 멘토: 학습 내용을 실제 프로젝트와 포트폴리오로 연결
- 문제해결 코치: 전원, 배선, 부품, 코드, 설계 문제를 체크리스트로 분석
- 안전 관리자: 전기, 배터리, 발열, 공구, 분해 작업 안전 점검
- 성장 기록 관리자: 학습 로그, 실습 기록, 실패 원인, 개선점 정리
- 학습 아카이브 관리자: 수업, 질문, 답변, 실습 아이디어를 계층형 기록으로 요약

## 10. 팀 회의형 답변을 사용하는 방법

사용자 입력에 아래 표현이 포함되면 팀 회의형 답변이 실행됩니다.

- 팀 회의
- 여러 선생님 의견
- 학교팀 전체
- 다 같이 검토
- 전문가처럼 분석
- 프로젝트로 연결
- 로드맵까지
- 직접 만들 수 있게

팀 회의형 답변 순서:

1. 학습 설계자
2. 개념 교수
3. 원리 해석가
4. 실습 코치
5. 설계 엔지니어
6. 프로젝트 멘토
7. 안전 관리자
8. 성장 기록 관리자
9. 학습 아카이브 관리자
10. 학교팀 매니저 최종 정리

## 11. learning_archive 폴더 구조

학습 기록 저장이 켜져 있으면 답변 뒤에 학습 아카이브 관리자가 요약을 만들고 `learning_archive/` 아래에 Markdown 파일로 저장합니다.

예시:

```text
learning_archive/
└─ 전자기기_엔지니어링/
   ├─ 01_기초개념/
   ├─ 02_부품이해/
   ├─ 03_제품분석/
   ├─ 04_실습기록/
   ├─ 05_프로젝트/
   └─ 06_오답과_문제해결/
```

현재 구현은 사용자의 현재 학습 목표를 기준으로 자동 기록 폴더를 만들며, `utils/archive.py`의 `save_markdown_archive()`를 확장하면 분야별 세부 폴더 규칙을 더 정교하게 만들 수 있습니다.

## 12. 추후 확장 아이디어

- 사용자별 학습 상태 저장
- 이전 대화 기반 개인화 로드맵 생성
- 아카이브 분류를 JSON 구조로 받아 더 정교하게 저장
- 이미지 입력을 받아 제품 구조 분석
- 실습 체크리스트와 부품 구매 목록 생성
- 분야별 학교팀 프리셋 추가
- 웹 UI 또는 모바일 앱으로 확장

## 13. 웹 배포 버전

`web/worker.js`는 Cloudflare Workers에 배포할 수 있는 웹 버전입니다.

- `/`: 학교팀 AI 에이전트 웹 UI
- `/api/chat`: OpenAI API를 호출하는 채팅 엔드포인트
- `/health`: 배포 상태 확인용 엔드포인트

웹 버전은 Cloudflare Worker secret `OPENAI_API_KEY`가 설정되어 있어야 실제 AI 답변을 생성합니다.

웹 화면에는 다음 대시보드 기능이 포함되어 있습니다.

- 에이전트별 프로필 카드
- 에이전트별 캐릭터 프로필 이미지와 성격
- 질문에 맞는 에이전트가 각자 자기 역할의 말투로 답하는 대화형 응답
- 자료 시각화 큐레이터가 그래프, 사진, 참고자료, 검색 키워드 안내
- 질문마다 어떤 에이전트가 참여했는지 보여주는 활동 히스토리
- 자동 생성되는 요약 노트와 노트 열기 버튼
- 6개월 로드맵 기준 현재 진도 패널과 진도 산정 근거
- 브라우저 `localStorage` 기반 대화, 노트, 활동 히스토리, 진도 유지
- 입력창에서 Enter는 전송, Shift+Enter는 줄바꿈
- 에이전트 답변은 프로필 옆 말풍선으로 표시되고, 사용자 대화는 오른쪽 말풍선으로 표시

GitHub + Cloudflare 배포 준비 사항은 `DEPLOYMENT.md`를 참고하세요.
